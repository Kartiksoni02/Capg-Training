JavaScriptEngine :- 

FLOW :- JS code > Parser(Tokenization) > AST > Interpreter ( byte code and hot code) > Profiler > JIT > optimized code

JS Code: The source code is first broken down into smaller chunks called tokens (e.g., identifiers, operators) through a process called tokenization.

Parser: The parser checks the tokens for syntax errors and organizes them into a tree structure.

parser :- first the code will be converted in chunks and then parse the chunks into AST format (Abstract Syntax Tree) tree like structure
then these chunks goes to interpreter then code is converted to bytecode (the code which is neither understandable by humans nor machine)
and JIT(just in time) compiler is also there so after interpreter, the code will go to JIt which is having profiler

AST (Abstract Syntax Tree): This is a hierarchical representation of the program's structure, allowing the engine to understand the logic before conversion.

Interpreter: The interpreter (such as V8's Ignition) converts the AST into Bytecode, an intermediate low-level format.
Bytecode: This code is executed line-by-line for a fast startup.
Hot Code: Frequently called functions or resource-intensive segments are identified during this phase.
HotCode:- the code which is repreating again and again like a loop.

profiler:- gets chunks of code from compiler and it gives hot code to JIT compiler
Profiler: While the interpreter runs, a profiler monitors the code in real-time, collecting data on data types and identifying "hot" code paths.

JIT :- convert hot code into optimized code

Optimized Code: The JIT compiler generates highly efficient machine code tailored to the specific CPU.

> Promises are objects representing the eventual completion (or failure) of an asynchronous operation.
> can handle the outcome of these operations using three primary handler methods: .then(), .catch(), and .finally().

1) .then() : used to handle a fulfilled promise and access its successful result. 
2) .catch() : used to handle a rejected promise and manage errors.
3) .finally() : a function to be called when the promise is settled, regardless of whether it was fulfilled or rejected.
